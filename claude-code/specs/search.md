# 搜索与证据规范

## 触发条件

严格区分内部知识与外部知识。不得基于通用内部知识推测，不确定时必须明确告知用户。即使拥有常识性知识 (如 fastapi 用法)，仍必须依据最新搜索结果或官方文档实现。

## 执行规则

- 使用 `groksearch` MCP 工具执行搜索
- 独立搜索请求并行执行，仅依赖关系时顺序执行
- 评估结果质量: 相关性、来源可信度、跨源一致性、完整性。存在缺口时补充搜索

## 信源质量标准

- 关键事实主张需 ≥2 个独立信源支持，单一信源时明确标注局限
- 冲突信源: 展示双方证据，评估可信度与时效性，标明较强证据或声明未解决分歧
- 经验性结论需标注置信度 (High/Medium/Low)
- 引用格式: [作者/机构, 年份/日期, 章节/URL]，严禁编造引用
